# シグナル入力ルール案

## 目的
実データでのスコア精度を上げるため、`contact_form_url` と `industry_fit` の入力ルールを定義する。

この文書では、**誤情報回避を優先しつつ、営業現場で再現しやすい最小ルール** を置く。

## 1. `contact_form_url` の入力ルール

### 基本方針
- 企業公式サイト起点で確認できた問い合わせ導線のみ入力する
- 問い合わせ導線が確認できない場合は空欄にする
- 電話番号だけがある場合は `contact_form_url` を埋めない
- SNSプロフィールや予約サイトだけでは `contact_form_url` としない

### 入力してよいケース
- 企業公式サイト内の `お問い合わせ` ページ
- 企業公式サイト内の `contact` ページ
- 企業公式サイト内リンクから遷移する外部フォーム
  - 例: Google Form, form.run, HubSpot, RESERVA

### 入力しないケース
- 企業公式サイトが確認できていない
- SNSリンクしかない
- 食べログ、ぐるなび、じゃらん等の外部予約ページしかない
- メールアドレスしか見つかっていない

### 迷ったときの扱い
- 「問い合わせフォームか予約フォームか」が曖昧な場合
  - 宿泊、飲食、来店予約中心の導線は `contact_form_url` にしない
- 企業公式サイトからの遷移か不明な場合
  - 空欄にする

### 実務上の判定優先順位
1. 公式サイト内の専用問い合わせページ
2. 公式サイト内リンク先の外部フォーム
3. 空欄

## 2. `industry_fit` の入力ルール

### 基本方針
- `industry_fit` は、**営業対象としての業種適合度** を表す
- 推測で入れず、ソースから確認できる業種情報を根拠にする
- 初回は `0`, `0.5`, `1` の3段階で扱う

### スコア定義
- `1`
  - まいぷれ事業との親和性が高い本命候補
  - 地域の中小店舗・中小企業を既に顧客として持ち、まいぷれを横展開しやすいと説明できる
- `0.5`
  - 親和性はあるが、`1` と言い切るには根拠が一段弱い
  - 営業余地はあるが、本命候補とまでは断定しない
- `0`
  - 適合根拠が確認できない
  - 判断材料不足

### `industry_fit=1` の厳しめ基準
次をすべて満たす場合にのみ `1` を付ける。

1. 企業の主力事業が公開情報で明確に確認できる
2. 地域の中小店舗・中小企業を顧客として持っていることが公開情報で確認できる
3. その顧客に対して、営業・提案・支援を行う立場であることが確認できる
4. その支援内容が、まいぷれ事業の提案と明確につながると説明できる
5. 上記の判断を、会社名の印象ではなく事業内容ベースで説明できる
6. 少しでも迷いがある場合は `1` にせず、`0.5` 以下に留める

### `industry_fit=0.5` の位置づけ
- 親和性はあるが、`1` とまでは言えない場合に付ける
- 典型例
  - 顧客対象は近そうだが、公開情報で明言されていない
  - 支援内容は近いが、まいぷれとの接続がやや弱い
  - 本命候補とまでは言い切れない

### `industry_fit=1` 判定で重視する公開記述
- 顧客対象
  - 例: 地域の中小企業向け、地元店舗向け、小売店・飲食店向け、地域事業者向け
- 支援内容
  - 例: 販促支援、集客支援、広告提案、Web集客支援、店舗運営支援

### `industry_fit=1` にしにくい記述
- 法人向けサービス
- 地域密着
- 幅広い業種に対応
- トータルサポート

### 入力根拠として使ってよい情報
- 企業公式サイトの事業内容
- 商工会議所や団体ページの業種・紹介文
- 会社概要ページの事業説明

### 入力根拠として弱いもの
- 会社名の印象だけ
- なんとなく近そうという推測
- 口コミサイトのカテゴリだけ

### 迷ったときの扱い
- 根拠が薄い場合は `0`
- 「少し近い」程度なら `0.5`
- 明確に適合すると読めるときだけ `1`

## 3. 初回の運用ルール
- `contact_form_url`
  - 見つからなければ空欄でよい
  - 無理に埋めない
- `industry_fit`
  - 業種判断の根拠が確認できた企業から少しずつ入れる
  - 未確認の企業は `0`

## 4. スコア入力の優先順
1. `detail_source_url`
2. `contact_form_url`
3. `local_focus`
4. `network_affinity`
5. `contactability`
6. `industry_fit`

## 5. 3地域の現状への当てはめ
- 岡崎市
  - まず `contact_form_url` の補完余地がある企業から進める
  - `industry_fit` は公式サイトの事業内容確認が必要
- 津山市
  - `contact_form_url` は未入力が多いので、まず公式サイト内導線の有無を確認する
  - `industry_fit` は飲食・宿泊・交通などで根拠化しやすい
- 高山市
  - `contact_form_url` は未入力のままでよいが、補完余地はある
  - `industry_fit` は観光、金融、建設、交通などで差を付けやすい
