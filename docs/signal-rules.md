# シグナル入力ルール案

## 目的
実データでのスコア精度を上げるため、`contact_form_url` と `industry_fit` の入力ルールを定義する。

この文書では、**誤情報回避を優先しつつ、営業現場で再現しやすい最小ルール** を置く。

## 1. `contact_form_url` の入力ルール

### 基本方針
- 企業公式サイト起点で確認できた問い合わせ導線のみ入力する
- 問い合わせ導線が確認できない場合は空欄にする
- 電話番号だけがある場合は `contact_form_url` を埋めない
- SNSプロフィールや予約サイトだけでは `contact_form_url` としない

### 入力してよいケース
- 企業公式サイト内の `お問い合わせ` ページ
- 企業公式サイト内の `contact` ページ
- 企業公式サイト内リンクから遷移する外部フォーム
  - 例: Google Form, form.run, HubSpot, RESERVA

### 入力しないケース
- 企業公式サイトが確認できていない
- SNSリンクしかない
- 食べログ、ぐるなび、じゃらん等の外部予約ページしかない
- メールアドレスしか見つかっていない

### 迷ったときの扱い
- 「問い合わせフォームか予約フォームか」が曖昧な場合
  - 宿泊、飲食、来店予約中心の導線は `contact_form_url` にしない
- 企業公式サイトからの遷移か不明な場合
  - 空欄にする

### 実務上の判定優先順位
1. 公式サイト内の専用問い合わせページ
2. 公式サイト内リンク先の外部フォーム
3. 空欄

## 2. `industry_fit` の入力ルール

### 基本方針
- `industry_fit` は、**営業対象としての業種適合度** を表す
- 推測で入れず、ソースから確認できる業種情報を根拠にする
- 初回は `0`, `0.5`, `1` の3段階で扱う

### スコア定義
- `1`
  - 営業対象として明確に適合している
  - 企業紹介や公式サイトで、対象業種・提供サービスがはっきり確認できる
- `0.5`
  - 関連業種ではあるが、適合がやや弱い
  - 主力事業が完全一致ではないが、営業余地がありそう
- `0`
  - 適合根拠が確認できない
  - 判断材料不足

### 入力根拠として使ってよい情報
- 企業公式サイトの事業内容
- 商工会議所や団体ページの業種・紹介文
- 会社概要ページの事業説明

### 入力根拠として弱いもの
- 会社名の印象だけ
- なんとなく近そうという推測
- 口コミサイトのカテゴリだけ

### 迷ったときの扱い
- 根拠が薄い場合は `0`
- 「少し近い」程度なら `0.5`
- 明確に適合すると読めるときだけ `1`

## 3. 初回の運用ルール
- `contact_form_url`
  - 見つからなければ空欄でよい
  - 無理に埋めない
- `industry_fit`
  - 業種判断の根拠が確認できた企業から少しずつ入れる
  - 未確認の企業は `0`

## 4. スコア入力の優先順
1. `detail_source_url`
2. `contact_form_url`
3. `local_focus`
4. `network_affinity`
5. `contactability`
6. `industry_fit`

## 5. 3地域の現状への当てはめ
- 岡崎市
  - まず `contact_form_url` の補完余地がある企業から進める
  - `industry_fit` は公式サイトの事業内容確認が必要
- 津山市
  - `contact_form_url` は未入力が多いので、まず公式サイト内導線の有無を確認する
  - `industry_fit` は飲食・宿泊・交通などで根拠化しやすい
- 高山市
  - `contact_form_url` は未入力のままでよいが、補完余地はある
  - `industry_fit` は観光、金融、建設、交通などで差を付けやすい
